## Headings \<- this one is h2 

To create a heading, add one to six `#` symbols before your heading text.
The number of `#` you use will determine the hierarchy level and typeface size of the heading.

```markdown
# H1
## H2
### H3
#### H4
```

### This is an H3 header
#### This is an H4 header - last one styled

### Links

The simplest one - just directly use url, like https://ajur.pl/ or www.ajur.pl - this will become clickable links, if they start with `http` or `www`.

To create a linked text, use the following syntax: `[link text](URL)`. Like link to [my home page](https://ajur.pl/).

```markdown
[My home page](https://ajur.pl/)
```

Section links (To headings) work as well, just use `#` followed by the heading text in lowercase, with spaces replaced by hyphens. For example link to [go to top](#markdown-cheat-sheet):

```markdown
[go to top](#markdown-cheat-sheet)
```

Relative links work as well, just remember to use relative/absolute url path, not content file path. For example: relative link to [links page](../links/), and absolute linkt to [first article](/articles/first-article/):

```markdown
[links page](../links/)
[First Article](/articles/first-article/)
```

Links url can also be added as reference, which is useful for long urls or when you want to reuse the same url multiple times. You can define a reference link like this: `[reference name]: URL`, and then use it in your text like this: `[link text][reference name]`. For example:
Or even shorter `[ref-name][]` if reference name is the same as link text:

```markdown
Either [Google browser][Google]
Or this [Google][]
Will link to the same place.

[Google]: https://www.google.com
```

### Footnotes

Footnotes syntax is similar to reference links, but with a caret `^` instead of square brackets. You can define a footnote like this: `[^1]: Footnote content`, and then reference it in your text like this: `[^1]`. For example:

```markdown
This is some text with a footnote reference.[^1]
[^1]: This is the content of the footnote.
```

Rendered footnote[^1] will appear as a superscript number, and the content will be displayed at the bottom of the page.

[^1]: This is the content of the footnote.

### Blockquotes

To create a blockquote, add a `>` in front of a paragraph. You can also nest blockquotes by adding additional `>` symbols.

> This is a blockquote.
> It can span multiple lines and paragraphs.
>
> > This is a nested blockquote.

```markdown
> This is a blockquote.
> It can span multiple lines and paragraphs.
>
> > This is a nested blockquote.
```

### Code blocks

To create a code block, wrap your code in triple backticks (` ``` `) and optionally specify the language for syntax highlighting.
Use more backticks if your code contains triple backticks.

Note that usual markdown syntax of indentation won't work, as Astro uses [GHF][ghf-page] for rendering.

[ghf-page]: https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/about-writing-and-formatting-on-github

````markdown
```javascript
function greet(name) {
  return Math.PI; // 3.14...
}
```
````

Astro uses [Shiki](https://github.com/shikijs/shiki) for syntax highlighting, which supports a wide range of languages and themes.

I've also added custom plugin to shiki, witch allows passing in css class to code blocks.
Thats usefull for adding utility classes. Most notably:
- `.full-bleed` for making code blocks span the full width of viewport.
- `.soft-bleed` for making code blocks wide enough to show all code without horizontal scrolling (if fits in viewport).

To add them, just add class (with dot) after language name:

````markdown
```javascript .full-bleed
// code here will be full bleed
```
````

## Images

### Basic markdown syntax

Basic syntax for adding images is similar to links, but with an exclamation mark at the beginning: `![alt text](image URL)`.
Local images paths are relative to current md file.

```markdown
![Nature pic](https://static.photos/nature/320x180/115)
![Mountains](./img-local-to-content-mdx.webp)
```

![Mountains](./md-test-embed-img.webp)

### Astro `<Image>` component

Astro provides an `<Image>` and `<Picture>` [components](https://docs.astro.build/en/guides/images/#astro-components-for-images) that offers advanced features like automatic optimization, lazy loading, and support for various image formats.

These components are available only in `.astro` or `.mdx` files, and not in regular markdown files.

To use it, you need to import the component, and image (unless its external or in public folder), than use Image component in your mdx:

```markdown .soft-bleed
import { Image } from 'astro:assets';
import test_img from './img-local-to-content-mdx.webp';

<Image src="{test_img}" alt="Mountains" width="100px" />
```

<Image src={test_img} alt="Mountains" width="100px" />

It supports even [more attributes](https://docs.astro.build/en/reference/modules/astro-assets/#image-), but also accepts any valid HTML attributes, so you can add custom classes, styles, data attributes, etc.

```markdown .soft-bleed
<Image src={test_img} alt="Mountains" layout='full-width' style="height: 10vh" />
```

<Image src={test_img} alt="Mountains" layout='full-width' style="height: 10vh" />

> [!tip]
> I've set some global defaults for images, in particular:
> - `responsiveStyles: true` to make images responsive by default.
> - `layout: 'constrained'` to make images scale down to fit their container
>   - `full-width` will enlarge image to fit parent container.
>   - other values are very specific, [check css docs](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Properties/object-fit).
> - `objectPosition: 'center'` to make sure images cover their container without distortion and are centered. [More options](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Properties/object-position) are for really specific use.
> - also, as objectPosition only defines position within container, I've added centering styles to all images by default.

## Tables

To create a table, use pipes `|` to separate columns and hyphens `-` to create the header row. You can also align text in columns using colons `:`.

```markdown
| Syntax      | Description |
| ----------- | ----------- |
| Header      | Title       |
| Paragraph   | Text        |

| Left Aligned | Center Aligned | Right Aligned |
| :--- |:---:| ---:|
| Left         | Center         | Right         |
```

| Syntax          | Description          |
| --------------- | -------------------- |
| Header          | Title                |
| Paragraph       | Text                 |
| Bit longer text | to see how it breaks |

| Left Aligned         |                       Center Aligned                        |                   Right Aligned |
| :------------------- | :---------------------------------------------------------: | ------------------------------: |
| Left                 |                           Center                            |                           Right |
| Some extra long text | to ensure that table is responsive and doesn't break layout | even if it has to be scrollable |
